## Directory Purposes

**`src/proof_cli/`:**
- Purpose: all application code lives here.
- Contains: CLI wiring, storage, domain models, retrieval, review, export, publication, governance, collaboration, memory, verification, and testable workflow helpers.
- Key files: `cli.py`, `commands.py`, `storage.py`, `proof_state.py`, `theorems.py`, `references.py`, `retrieval.py`, `memory.py`, `collaboration.py`, `snapshot.py`, `services.py`.

**`tests/`:**
- Purpose: unit and workflow tests for each module boundary.
- Contains: one `test_*.py` file per major module or workflow area.
- Key files: `test_cli.py`, `test_storage.py`, `test_proof_state.py`, `test_theorems.py`, `test_retrieval.py`.

**`.planning/`:**
- Purpose: roadmap, milestones, and codebase maps for GSD workflow.
- Contains: project planning docs and phase artifacts.
- Key files: `PROJECT.md`, `ROADMAP.md`, `REQUIREMENTS.md`, `STATE.md`, `.planning/codebase/*.md`.

**`.agents/skills/`:**
- Purpose: local skill guidance for repository-specific workflows.
- Contains: `proof-cli/SKILL.md`.
- Key files: `.agents/skills/proof-cli/SKILL.md`.

## Key File Locations

**Entry Points:**
- `pyproject.toml`: defines `proof = "proof_cli.cli:app"`.
- `src/proof_cli/cli.py`: Typer app and nested command groups.

**Configuration:**
- `pyproject.toml`: build system, dependencies, pytest config, entrypoint.
- `.planning/config.json`: GSD planning configuration.

**Core Logic:**
- `src/proof_cli/commands.py`: command orchestration and formatting.
- `src/proof_cli/proof_state.py`: state transitions, snapshot assembly, history.
- `src/proof_cli/storage.py`: SQLite and file-backed persistence.
- `src/proof_cli/theorems.py`: theorem contract lifecycle.
- `src/proof_cli/retrieval.py`: search/ranking over local and external candidates.

**Testing:**
- `tests/test_*.py`: co-located by feature area, one module per concern.

## Naming Conventions

**Files:**
- Module-per-domain naming with short singular nouns: `theorems.py`, `references.py`, `snapshot.py`, `review.py`.
- Tests mirror module names: `test_theorems.py`, `test_storage.py`.

**Functions:**
- CLI orchestration functions use the `cmd_*` prefix in `commands.py`.
- Lower-level state mutations use verb phrases such as `add_theorem`, `store_contract`, `build_snapshot`, `record_memory`.

**Variables:**
- Workspace objects are named `store`, `state`, `snapshot`, `contract`, `reference`, `obligation`, and `blocker`.

**Types:**
- Domain records are Pydantic `BaseModel` classes with PascalCase names.
- Enums encode workflow states and are named as nouns or adjective phrases, such as `TheoremStatus` and `ReviewGovernanceState`.

## Where to Add New Code

**New CLI command:**
- Primary code: `src/proof_cli/cli.py`
- Orchestration: `src/proof_cli/commands.py`
- Tests: `tests/test_cli.py` or the feature-specific test file

**New proof-state mutation:**
- Primary code: `src/proof_cli/proof_state.py`
- Persistence: `src/proof_cli/storage.py`
- Tests: `tests/test_proof_state.py`, `tests/test_storage.py`

**New theorem/reference behavior:**
- Implementation: `src/proof_cli/theorems.py` or `src/proof_cli/references.py`
- Storage hooks: `src/proof_cli/storage.py`
- Tests: `tests/test_theorems.py`, `tests/test_references.py`

**New retrieval/ranking logic:**
- Implementation: `src/proof_cli/retrieval.py`
- Workspace-facing wrapper: `src/proof_cli/services.py`
- Tests: `tests/test_retrieval.py`

**New collaboration or publication workflow:**
- Implementation: `src/proof_cli/collaboration.py` or `src/proof_cli/publication.py`
- Tests: `tests/test_collaboration.py`, `tests/test_publication.py`

**New memory/debug artifact:**
- Implementation: `src/proof_cli/memory.py` and related helpers in `src/proof_cli/debug_tasks.py`
- Tests: `tests/test_debug_tasks.py`, `tests/test_proof_state.py`
