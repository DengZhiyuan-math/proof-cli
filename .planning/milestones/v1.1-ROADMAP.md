# Milestone v1.1: Collaborative Proof Infrastructure

**Status:** ✅ SHIPPED 2026-04-29
**Phases:** 6-9
**Total Plans:** 6

## Overview

v1.1 delivered collaborative proof infrastructure, publication-grade proof delivery, retrieval and diagnostic snapshots, and a real radial-system validation slice. The milestone kept the workflow CLI-first, auditable, and recovery-friendly while extending it into governed collaboration and project-aware retrieval.

## Phases

### Phase 6: Collaborative Proof Infrastructure

**Goal**: Validate multi-user collaborative proof development with explicit governance, provenance, review, branching, team libraries, reproducible handoff, and collaboration-aware policy.
**Depends on**: Phase 5
**Plans**: 2 plans

Plans:

- [x] 06-01: Collaboration layer with contributor/authorship schema, review records, comment threads, branches, collaboration policy storage, shared publication tracking, CLI reachability, and export/status visibility
- [x] 06-02: Exchange path with bundle export/import, handoff creation and inspection, and two-user collaboration validation

**Details:**
Phases 6-1 and 6-2 established the shared proof substrate used by the rest of the milestone. The collaboration workflow, bundle exchange, and handoff inspection paths were validated with real tests and preserved provenance through transfer.

### Phase 7: Publication-Grade Proof Pipelines

**Goal**: Turn internal proof-state, review history, and verification artifacts into publication-grade research delivery outputs.
**Depends on**: Phase 6
**Plans**: 2 plans

Plans:

- [x] 07-01: Publication workspace models, selective paper/supplement exports, and auditable snapshot/release plumbing
- [x] 07-02: Mixed-provenance publication validation on a real project slice

**Details:**
Phase 7 separated theorem trust from editorial readiness, added selective exports, and confirmed bundle round-tripping while preserving internal-only redaction boundaries.

### Phase 8: Retrieval and Snapshots

**Goal**: Make retrieval and session recovery project-aware.
**Depends on**: Phases 6 and 7
**Plans**: 1 plan

Plans:

- [x] 08-01: Structural-first retrieval, JSON diagnostics, and bounded diagnostic snapshots

**Details:**
Phase 8 made retrieval prioritize theorem state, obligations, blockers, memory, and graph adjacency before loose matching, and it bounded snapshots to recovery state plus the latest diagnostic report.

### Phase 9: Radial System Validation

**Goal**: Prove the new memory and graph layer is useful on a real unfinished proof section.
**Depends on**: Phase 8
**Plans**: 1 plan

Plans:

- [x] 09-01: Radial-system validation on a smaller theorem cluster with proof-logic partitioning

**Details:**
Phase 9 validated the workflow on a smaller theorem cluster instead of the full section, proving that the combined retrieval, diagnostics, snapshot, and exchange flow can clarify a real proof gap.

## Milestone Summary

**Key Decisions:**

- Keep collaboration explicit with authorship, review history, and governance states.
- Keep publication views selective so paper-facing output stays stricter than supplement and bundle outputs.
- Keep retrieval structural-first, with theorem state, obligations, blockers, memory, and graph adjacency ahead of loose matching.
- Keep snapshots bounded to recovery state plus the latest diagnostic report.
- Validate section-level proof work on a smaller theorem cluster organized by proof logic and complexity.

**Issues Resolved:**

- Completed the two-user collaboration and exchange path.
- Preserved trust boundaries through selective publication exports.
- Confirmed project-aware retrieval and diagnostics on real workspace state.
- Confirmed the radial-system workflow helps identify a small scalar-to-vector lift gap and a Jacquet bridge.

**Issues Deferred:**

- The v2 project-memory and explicit graph requirements remain deferred to the next milestone cycle.

**Technical Debt Incurred:**

- No blocking technical debt was accepted in the shipped milestone scope.
