# Milestone v1.2: Codex Command Routing

**Status:** ✅ SHIPPED 2026-04-30
**Phases:** 10-13
**Total Plans:** 4

## Overview

v1.2 turned Proof CLI into a real Codex-facing command surface. The milestone replaced weak skill-only triggering with a wrapper-backed workflow that can initialize workspaces, inspect state, route theorem mutations safely, diagnose degraded command states, and validate a realistic small theorem cluster end to end.

## Phases

### Phase 10: Command Invocation Surface

**Goal**: Make Codex treat `proof` commands as a stable invocation surface instead of a loose skill hint.
**Depends on**: Phases 8 and 9
**Plans**: 1 plan

Plans:

- [x] 10-01: Deterministic wrapper routing, guided catalog output, aligned skills, and read-only smoke coverage

**Details:**
Phase 10 introduced `proof codex` and `proof-codex` as the hard wrapper surface, aligned project-local and global `proof` skills around the same semantics, preserved machine-readable retrieval/analysis output, and established deterministic common-case root resolution.

### Phase 11: Mutation Routing and Root Resolution

**Goal**: Route mutating Proof CLI workflows safely through Codex with reliable workspace targeting.
**Depends on**: Phase 10
**Plans**: 1 plan

Plans:

- [x] 11-01: Theorem, obligation, blocker, and snapshot mutation routing with explicit root and persistence feedback

**Details:**
Phase 11 extended the wrapper into real theorem-state mutations, enforced stricter write-time root rules, and made underspecified commands fail gently with minimum-missing-detail guidance instead of dumping raw CLI help.

### Phase 12: Bootstrap and Wrapper Hardening

**Goal**: Make the command layer resilient when local configuration or executable setup is incomplete.
**Depends on**: Phase 11
**Plans**: 1 plan

Plans:

- [x] 12-01: Command-readiness diagnostics, degraded-state guidance, and canonical global-skill clarification

**Details:**
Phase 12 added `proof codex doctor`, surfaced readiness in the wrapper catalog, standardized advisory-only recovery messaging, and made the global `~/.codex/skills/proof/` skill the canonical production entry path.

### Phase 13: Codex E2E Validation

**Goal**: Prove the command-routing layer can drive a real miniature proof workflow from Codex.
**Depends on**: Phases 10, 11, and 12
**Plans**: 1 plan

Plans:

- [x] 13-01: Wrapper-first theorem-cluster validation workflow with `init`, mutations, retrieval, diagnostics, and snapshots

**Details:**
Phase 13 validated a realistic theorem-cluster workflow through the hardened wrapper, added `proof codex init`, and showed that the new command surface is materially more usable and less ambiguous than the earlier skill-only path.

## Milestone Summary

**Key Decisions:**

- Keep Proof CLI as the source of truth and execution engine; Codex gets a wrapper, not a second proof-state implementation.
- Keep read-only and mutating trust boundaries explicit through wrapper messaging and root diagnostics.
- Treat the global `proof` skill as canonical, and reframe project-local skills as development/debug aids.
- Prefer advisory diagnostics over automatic environment repair when command-layer prerequisites are missing.
- Judge success by realistic workflow usability, not only by parser-level smoke coverage.

**Issues Resolved:**

- Replaced ambiguous `$proof ...` skill triggering with deterministic wrapper-backed routing.
- Added safe theorem, obligation, blocker, and snapshot mutation flows from Codex.
- Made readiness failures visible and explainable without auto-installing tools.
- Validated an end-to-end command-driven theorem workflow closer to real research usage.

**Issues Deferred:**

- Richer interactive theorem-edit forms remain future work.
- Multi-workspace selection without manual root disambiguation remains future work.
- Structured execution logs and deeper plugin-level packaging remain future work.

**Technical Debt Incurred:**

- No blocking technical debt was accepted in the shipped milestone scope.
